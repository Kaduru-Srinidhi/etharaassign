# Team Task Manager - Complete Setup & Deployment Instructions

## 📋 Project Summary

**Team Task Manager** is a full-stack web application for collaborative project and task management with role-based access control.

### Key Features
- ✅ User authentication (Signup/Login with JWT)
- ✅ Project & team management
- ✅ Task creation, assignment & status tracking
- ✅ Kanban board (To Do → In Progress → Done)
- ✅ Dashboard with task overview
- ✅ Role-based access control (Admin/Member)
- ✅ Responsive UI with modern design

---

## 🏗️ Architecture

### Tech Stack
- **Backend**: Node.js + Express.js + PostgreSQL
- **Frontend**: React 18 + Vite + Axios
- **Authentication**: JWT tokens + bcryptjs
- **Deployment**: Railway

### Database Schema
- **Users**: Store user accounts with roles
- **Projects**: Projects created by users
- **Project Members**: Team members in projects with roles
- **Tasks**: Tasks assigned to team members

## 📦 Getting Started Locally

### Step 1: Prerequisites
```bash
# Install Node.js (v16+) from https://nodejs.org
# Install PostgreSQL from https://www.postgresql.org
# Verify installations:
node --version
npm --version
psql --version
```

### Step 2: Clone/Download Repository
```bash
cd ~/Desktop  # or your preferred location
# (The code is already in etharaassignment folder)
cd etharaassignment
```

### Step 3: Setup PostgreSQL Database
```bash
# Start PostgreSQL service (if not auto-started)
# macOS: brew services start postgresql
# Linux: sudo systemctl start postgresql
# Windows: Services > PostgreSQL > start

# Create database
psql -U postgres
postgres=# CREATE DATABASE team_task_manager;
postgres=# \q
```

### Step 4: Configure Backend
```bash
cd backend

# Copy environment template
cp .env.example .env

# Edit .env with your PostgreSQL credentials:
# DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/team_task_manager
# JWT_SECRET=your-super-secret-key-make-it-very-long-and-random
# NODE_ENV=development
# PORT=5000
```

### Step 5: Install & Start Backend
```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Expected output:
# Database initialized successfully
# Server running on port 5000
```

### Step 6: Configure Frontend (New Terminal)
```bash
cd frontend

# Copy environment template
cp .env.example .env

# .env already has correct settings:
# VITE_API_URL=http://localhost:5000
```

### Step 7: Install & Start Frontend
```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Visit: http://localhost:3000
```

### Step 8: Test the Application
1. **Create Account**
   - Click "Sign up"
   - Enter name, email, password
   - Submit

2. **Create Project**
   - On Dashboard, click "Create New Project"
   - Enter project name and description
   - Click "Create Project"

3. **Create Task**
   - Click on project from dashboard
   - Fill in task title, description, priority, due date
   - Assign to team member
   - Click "Create Task"

4. **Move Task**
   - Drag task from "To Do" to "In Progress" or "Done"
   - Or click arrow button to advance status

5. **Add Team Member**
   - In project's "Project Members" section
   - Enter user ID (get from another user's signup)
   - Select role (Member or Admin)
   - Click "Add Member"

## 🚀 Deployment on Railway

### Step 1: Create Railway Account
1. Visit https://railway.app
2. Sign up with GitHub (recommended)
3. Create new project

### Step 2: Deploy PostgreSQL
1. In Railway dashboard: "New" → "Database" → "PostgreSQL"
2. Railway auto-creates database with connection string
3. Copy the `DATABASE_URL` from PostgreSQL service variables
4. You'll use this in backend

### Step 3: Deploy Backend Service

1. **Create New Service**
   - In Railway: "New" → "GitHub Repo"
   - Authorize GitHub
   - Select this repository

2. **Configure Service**
   - Service Settings → Source
   - Set root directory: `backend`

3. **Set Environment Variables**
   - Click Variables tab
   - Add following variables:
     ```
     DATABASE_URL=<copy from PostgreSQL service>
     JWT_SECRET=generate-a-long-random-secret-here
     NODE_ENV=production
     PORT=5000
     ```

4. **Deploy**
   - Save variables
   - Railway auto-deploys on push
   - Wait for deployment to complete
   - Copy the deployed URL (e.g., `https://app-backend-prod.railway.app`)

### Step 4: Deploy Frontend Service

1. **Create New Service**
   - In Railway: "New" → "GitHub Repo"
   - Select this repository

2. **Configure Service**
   - Service Settings → Source
   - Set root directory: `frontend`

3. **Set Environment Variables**
   - Click Variables tab
   - Add following:
     ```
     VITE_API_URL=<your backend Railway URL>
     ```
   - Example: `VITE_API_URL=https://app-backend-prod.railway.app`

4. **Configure Build & Deploy**
   - In Deployments tab
   - Build command: `npm install && npm run build`
   - Start command: (leave empty, Railway handles it)
   - Railway will serve `dist` folder as static site

5. **Deploy**
   - Save configuration
   - Railway auto-deploys
   - Wait for completion
   - Get frontend URL from deployment

### Step 5: Verify Live Deployment

1. Visit your frontend URL (from Railway)
2. Create test account
3. Create project
4. Create task
5. Test all features

## 🔒 API Endpoints

### Authentication
- `POST /api/auth/signup` - Register new user
- `POST /api/auth/login` - Login user
- `GET /api/auth/me` - Get current user

### Projects
- `POST /api/projects` - Create project
- `GET /api/projects` - Get user's projects
- `GET /api/projects/:id` - Get project details
- `PUT /api/projects/:id` - Update project
- `DELETE /api/projects/:id` - Delete project
- `POST /api/projects/:id/members` - Add team member
- `GET /api/projects/:id/members` - Get project members

### Tasks
- `POST /api/tasks/:projectId/tasks` - Create task
- `GET /api/tasks/:projectId/tasks` - Get project tasks
- `PUT /api/tasks/:projectId/tasks/:taskId` - Update task
- `DELETE /api/tasks/:projectId/tasks/:taskId` - Delete task
- `GET /api/tasks/dashboard/overview` - Get dashboard data

## 🐛 Troubleshooting

### Local Development Issues

**Backend won't start**
```bash
# Error: port already in use
# Solution: Change PORT in .env or kill process on port 5000

# Error: Cannot connect to database
# Solution: Ensure PostgreSQL is running
psql -U postgres  # test connection
```

**Frontend shows blank page**
```bash
# Error: Cannot GET /
# Solution: Frontend not running, check terminal for errors

# Error: API errors in console
# Solution: Ensure backend is running and VITE_API_URL is correct
```

**CORS errors in console**
```bash
# Error: "Access to XMLHttpRequest has been blocked by CORS policy"
# Solution: Backend CORS is already configured, restart both servers
```

### Railway Deployment Issues

**Backend deployment fails**
- Check logs in Railway dashboard
- Verify DATABASE_URL format
- Ensure JWT_SECRET is set
- Check Node version compatibility

**Frontend can't reach backend**
- Verify VITE_API_URL is set correctly in frontend
- Check backend service is running
- Test backend URL directly in browser

**Static files not loading**
- Ensure `npm run build` completes successfully
- Check that `dist` folder is created
- Verify build command in Railway
